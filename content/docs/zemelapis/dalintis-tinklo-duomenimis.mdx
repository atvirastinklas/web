import { Tab, Tabs } from 'fumadocs-ui/components/tabs';

[Atviro Tinklo žemėlapis](/map) naudoja oficialų Meshtastic® MQTT serverį duomenims rinkti ir rodyti.  
Kad žemėlapyje matytųsi jūsų ar kitų aplinkinių įrenginiai, bent vienas įrenginys turi perduoti pagrindinio kanalo duomenis į MQTT.

Instrukcijos skiriasi pagal tai, ar įrenginys turi Wi-Fi / Ethernet ryšį:

<Tabs items={['Turi Wi-Fi / Ethernet', 'Neturi interneto ryšio']} defaultIndex={1}>
  <Tab value="Turi Wi-Fi / Ethernet">
    Jei įrenginys palaiko **Wi-Fi arba Ethernet** ryšį, galite lengvai sukonfigūruoti duomenų siuntimą.  
    Vadovaukitės [oficialiomis instrukcijomis](https://meshtastic.org/docs/configuration/module/mqtt/#connect-to-the-default-public-server) ir įveskite šiuos nustatymus:


  | Nustatymas                | Reikšmė               |
  |---------------------------|-----------------------|
  | Enabled                   | `true` (įjungta)      |
  | MQTT server address       | `mqtt.meshtastic.org` |
  | MQTT username             | `meshdev`             |
  | MQTT password             | `large4cats`          |
  | Encryption enabled        | `true` (įjungta)      |
  | JSON enabled              | `false` (išjungta)    |
  | TLS enabled               | `false` (išjungta)    |
  | Root topic                | `msh/EU_868/LT`       |
  | MQTT Client Proxy enabled | `false` (išjungta)    |
  </Tab>

<Tab value="Neturi interneto ryšio">
  Jei įrenginys **neturi tiesioginio interneto ryšio**, duomenis galima perduoti per mobilųjį telefoną ar naršyklę.  
  Tačiau tai nėra patikimas ar ilgalaikis sprendimas - toks būdas tinkamas tik testavimui.  
  Plačiau apie šį nustatymą skaitykite čia: [„MQTT Client Proxy enabled“](https://meshtastic.org/docs/configuration/module/mqtt/#client-proxy-enabled).
</Tab>
</Tabs>
