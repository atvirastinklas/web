[Atviro Tinklo žemėlapis](/map) renka ir rodo duomenis naudodamas **MQTT serverį**.  
Kad žemėlapyje matytųsi jūsų ar aplinkinių įrenginiai, bent vienas iš jų turi siųsti pagrindinio kanalo duomenis į MQTT.

## Pagrindinis būdas: Wi-Fi arba Ethernet
Jei įrenginys palaiko **Wi-Fi** ar **Ethernet** ryšį, rekomenduojame naudoti šį metodą.  
Mūsų bendruomenė naudoja **savo MQTT serverį**, o jei jis būtų nepasiekiamas, galima naudoti oficialų **Meshtastic®** serverį.

Vadovaukitės [oficialiomis instrukcijomis](https://meshtastic.org/docs/configuration/module/mqtt/#connect-to-the-default-public-server) ir įveskite šiuos nustatymus:

| Nustatymas                | Reikšmė                  |
|---------------------------|--------------------------|
| Enabled                   | `true` (įjungta)         |
| MQTT server address       | `mqtt.atvirastinklas.lt` |
| MQTT username             | `vartai`                 |
| MQTT password             | `mazgas`                 |
| Encryption enabled        | `true` (įjungta)         |
| JSON enabled              | `false` (išjungta)       |
| TLS enabled               | `false` (išjungta)       |
| Root topic                | `msh/EU_868/LT`          |
| MQTT Client Proxy enabled | `false` (išjungta)       |

## Alternatyva: naudojant mobilųjį įrenginį
Jei jūsų įrenginys **neturi tiesioginio interneto ryšio**, duomenis galima perduoti per mobilųjį telefoną.  
Tačiau toks ryšys yra **nepatikimas ir netinka ilgalaikiam naudojimui** - rekomenduojame jį taikyti tik testavimo metu.

Plačiau apie šį nustatymą skaitykite čia: [„MQTT Client Proxy enabled“](https://meshtastic.org/docs/configuration/module/mqtt/#client-proxy-enabled).
